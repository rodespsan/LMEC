Introducción

Actualmente el Laboratorio de Mantenimiento de Equipo de Cómputo cuenta con un sistema de control de ordenes de servicio, sin embargo, este sistema ya no abarca las necesidades actuales del departamento y no es posible actualizarlo. Sólo se cuenta con una copia del sistema, el cual está instalado en una estación de trabajo ordinaria, lo que impide el acceso de múltiples usuarios al sistema, así como el riesgo que conlleva almacenar información dentro de una computora ordinaria. Adicionalmente, el sistema solo se ejecuta en el sistema operativo Windows XP, y no es posible actualizarlo. Debido a todo lo anterior, el sistema se considera obsoleto.

Este proyecto pretende desarrollar el Sistema de Control de Mantenimiento de Equipo de Cómputo es proveer un sistema al Laboratorio de Mantenimiento de Equipo de Cómputo, que automatice y facilite procesos internos, reduzca tiempos de respuesta e incremente la calidad del servicio y la satisfacción del usuario.

El sistema debe ajustarse a las necesidades actuales del Laboratorio de Mantenimiento de Equipo de Cómputo así como, ser accesible por múltiples usuarios desde múltiples plataformas, fácil de actualizar y mejorar, y que garantice la integridad, confidencialidad y disponibilidad de la información.

  • El sistema debe ser multiusuario.
  • El sistema debe ser accesible.
  • El sistema debe ser multiplataforma.
  • El sistema debe ser seguro.
  • El sistema se debe ajustar a las necesidades actuales del Laboratorio.

Los casos de uso son una lista de pasos que definen la interacción de los usuarios con el sistema para lograr un objetivo particular.

Los casos de uso se encuentran descritos en los documentos de Conjunto de Casos de Uso y Casos de Uso.

Requerimientos No Funcionales

La interfaz del usuario debe ser fácil de usar, intuitiva para que al usuario no le cueste tanto trabajo el cambio al nuevo sistema.

  • Usabilidad
    • Los usuarios necesitan entender fácilmente el sistema para usarlo.
    • El sistema debe ser intuitivo y la interfaz homogenea para que sea fácil de aprender y recordar.
  • Disponibilidad
    • El sistema debe estar disponible 24/7, excluyendo días de mantenimiento programados al sistema.
    • El sistema debe estar accesible desde cualquier sistema operativo usando los navegadores más comunes (Internet Explorer, Firefox, Chrome) en sus versiones recientes.
  • Mantenibilidad
    • Se dispone del código fuente para posteriores actualizaciones.
    • Utiliza estándares de codificación.
    • Tiene la documentación técnica necesaria para ser actualizado fácilmente por terceras personas.
  • Seguridad
    • El sistema debe estar protegido contra los diez ataques de seguridad a aplicaciones Web más comunes descritos por la OWASP.

Requerimientos de Entorno

Entorno tecnológico

Se requiere que sea un sistema en una arquitectura cliente-servidor para que sea accesible por múltiples usuarios remotamente.

Se requiere que el sistema pueda ser usado desde cualquier sistema operativo, usando los navegadores más comunes (Internet Explorer, FireFox y Chrome) en sus versiones más recientes.

Se ha decidido que sea un sistema web para que pueda cubrir el requerimiento anterior y además sea fácilmente actualizable y mantenible.


Entorno físico

Se requiere que la aplicación y la base de datos se encuentre resguardada, para esto se ha determinado que será ubicada en la Intranet en el área de servidores de la Facultad, la cual cuenta con aire acondicionado las 24 horas del día. Los servidores se encuentran funcionando 24/7 a excepción de los tiempos de mantenimiento generalmente realizados en períodos vacacionales. El área cuenta con UPS de respaldo en caso de cortes de energía de hasta de 20 minutos.

El servidor Intranet cuenta con:

  • doble fuente de poder para evitar pérdida de corriente.
  • configuración de arreglo de disco duro RAID 1.
  • sistema de respaldo diario a cinta.
