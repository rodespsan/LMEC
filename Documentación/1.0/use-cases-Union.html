CU-11: Crear cliente

Resumen: El actor crea un cliente con los campos solicitados.
Prioridad: Esencial
Frecuencia de Uso: Algunas veces
Actores Directos:
Administrador del sistema.
Recepcionista.
Prerequisitos:
Se debe haber iniciado sesión con un usuario con rol de administrador o recepcionista.
Debe existir al menos un tipo de cliente activo.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo de clientes y selecciona la opción crear cliente.
  2. El sistema muestra el formulario para capturar los datos del cliente.
  3. El actor captura los campos del formulario:

    1. Nombre de cliente (EA1)
    2. Tipo de cliente
    3. Dependencia
    4. Dirección del cliente
    5. Nombre de contacto (EA2)
    6. Correo electrónico(EA3)
    7. Teléfono de oficina
    8. Teléfono celular
    9. Teléfono de casa
    10. Extensión de oficina
    11. Activo
  4. El actor envia el formulario.
  5. El sistema guarda la información capturada. (EA4)
  6. El sistema despliega la información capturada.
Extensiones del Escenario Alternativo:
  1. El nombre de cliente no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guarda el cliente.
  2. El nombre de contacto no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guarda el cliente.
  3. El correo electrónico no puede ser vacío o únicamente contener caracteres vacíos y debe ser un formato de correo válido. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guarda el cliente.
  4. Si ocurre algún error al enviar el formulario, no se guarda el cliente, ni se redirecciona a la vista donde se muestra la información ingresada.
Notas y Preguntas
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo, el cliente no se visualiza para otros usuarios del sistema pero existe el proceso de activación.
  • Los campos(Nombre, Tipo de cliente, Nombre de contacto, Correo electrónico) son obligatorios y los campos (Dependencia, Dirección, Teléfono de oficina, Extension de oficina, Teléfono celular, Teléfono de casa, Activo) no son obligatorios.

  2. El nombre de contacto no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados.

  4. Si ocurre algún error al enviar el formulario, no se guardan los cambios realizados en los datos del cliente, ni se redirecciona a la vista donde se muestra la información actualizada.

Notas y Preguntas
  • Los campos(Nombre, Tipo de cliente, Nombre de contacto, Correo electrónico) son obligatorios y los campos (Dependencia, Dirección, Teléfono de oficina, Teléfono celular, Teléfono de casa, Extension de oficina, Activo) no son obligatorios.
