CU-11: Crear cliente

Resumen: El actor crea un cliente con los campos solicitados.
Prioridad: Esencial
Frecuencia de Uso: Algunas veces
Actores Directos:
Administrador del sistema.
Recepcionista.
Prerequisitos:
Se debe haber iniciado sesión con un usuario con rol de administrador o recepcionista.
Debe existir al menos un tipo de cliente activo.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo de clientes.
  2. El actor selecciona la opción crear cliente.
  3. El sistema muestra el formulario para capturar los datos del cliente.
  4. El actor captura los campos del formulario:
    1. Nombre de cliente (EA1)
    2. Tipo de cliente
    3. Dependencia
    4. Dirección
    5. Nombre de contacto (EA2)
    6. Correo electrónico(EA3)
    7. Teléfono de oficina
    8. Teléfono celular
    9. Teléfono de casa
    10. Extensión de oficina
    11. Activo
  5. El actor envia el formulario.
  6. El sistema despliega la información capturada. (EA4)
Extensiones del Escenario Alternativo:
  1. El nombre de cliente no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardará el cliente.
  2. El nombre de contacto no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente.
  3. El correo electrónico no puede ser vacío o únicamente contener caracteres vacíos y debe ser un correo válido, de lo contrario el sistema desplegará un mensaje informando el error y no guardará el cliente.
  4. Si ocurre algún error al enviar el formulario, no se guardará el cliente, ni redireccionará a la otra vista donde se muestra la información ingresada.
Notas y Preguntas
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo se le impide al usuario el acceso al sistema pero existe el proceso de activación.
  • Los campos(Nombre, Tipo de cliente, Nombre de contacto, Correo electrónico) son obligatorios y los campos (Dependencia, Dirección, Teléfono de oficina, Extension de oficina, Teléfono celular, Teléfono de casa, Activo) no son obligatorios.

En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados. (Cambiar el tiempo de los verbos)

  6. El sistema desactiva el cliente.

  1. Si el actor cancela la acción, el sistema no desactiva el cliente.

CU-17: Buscar clientes

Resumen: El actor busca clientes con caracteristicas especificas.
Prioridad: Esencial
Frecuencia de Uso: Algunas veces
Actores Directos:
Administrador del sistema.
Recepcionista.
Prerequisitos:
Se debe haber iniciado sesión con un usuario con rol de administrador o recepcionista.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo de clientes y selecciona la opción administrar clientes.
  2. El sistema despliega la lista de clientes con los campos:

    1. Id
    2. Nombre
    3. Tipo de cliente
    4. Nombre de contacto
    5. Dependencia
    6. Activo

  3. El actor captura los criterios de búsqueda.
  4. El sistema despliega los clientes que coincidan con los criterios de búsqueda (EA1).
Extensiones del Escenario Alternativo:
  1. Si no concide algún cliente con los criterios de búsqueda, el sistema despliega un mensaje indicando que no se encontraron resultados.

Notas y Preguntas
  • Se realiza una paginación automática de 20 registros por defecto y el actor puede cambiar el tamaño de la paginación.
  • Se permite ordenar los clientes a través de los campos(Id, Nombre, Tipo de cliente, Dependencia, Activo).
