CU-00: Crear usuario

Resumen: El actor navega al módulo de usuarios y crea el usuario con los campos solicitados.
Prioridad: Esencial
Frecuencia de Uso: Rara vez
Actores Directos:
Administrador del sistema
Prerequisitos:
 Se debe haber iniciado sesión con un usuario con rol de administrador.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo Usuarios.
  2. El actor ingresa a la opción crear usuario.
  3. El sistema muestra el formulario para capturar los datos del usuario. (AE1)
  4. El actor captura los campos del formulario:
    1. Usuario (EA2,EA3)
    2. Contraseña (EA4)
    3. Confirmar contraseña (EA5)
    4. Nombres (EA6)
    5. Apellidos (EA7)
    6. Correo electrónico(EA8)
    7. Roles (EA9)
    8. Activo
  5. El actor envia el formulario.
  6. El sistema despliega la información capturada. (EA10)
Extensiones del Escenario Alternativo:
  1. Si  no existen roles activos el sistema no permitirá visualizar el formulario y emitirá una advertencia.
  2. El usuario no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 3 y 20, respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardará el usuario.
  3. Si el usuario ya existe no se guardará, se regresará al formulario de captura y se desplegará el mensaje de error.
  4. La contraseña no puede ser vacía o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 8 y 20, respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente.
  5. Si confirmar contraseña es vacía, únicamente contiene caracteres vacíos o no coincide con la contraseña, el sistema no guardará el usuario e indicará el error correspondiente en el formulario.
  6. Los nombres no pueden ser vacíos o únicamente contener caracteres vacíos, de no cumplir lo requerido, el sistema desplegará el mensaje y no guadará el usuario.
  7. Los apellidos no pueden ser vacíos o únicamente contener caracteres vacíos, de no cumplir lo requerido, el sistema desplegará el mensaje y no guadará el usuario.
  8. El correo electrónico no puede ser vacío o únicamente contener caracteres vacíos y debe ser un correo válido, de lo contrario el sistema desplegará un mensaje informando el error y no guardará el usuario.
  9. Si no se selecciona al menos un rol, el sistema no guardará el usuario e indicará el error en el formulario.
  10. Si ocurre algún error al enviar el formulario, el usuario no se guardará, ni redireccionará a la otra vista donde se muestra la información capturada.
Notas y Preguntas:
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo se le impide al usuario el acceso al sistema pero existe el proceso de activación.
  • La contraseña deberá encriptarse en la base de datos usando MD5.

CU-01: Visualizar usuario

Resumen: El actor visualiza la información de un usuario.
Prioridad: Esperado
Frecuencia de Uso: Rara vez
Actores Directos:
 Administrador del sistema
Prerequisitos: Se debe haber iniciado sesión con un usuario con rol de administrador.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo Usuarios
  2. Existen tres formas de acceder a esta vista.
    1. El actor accede automáticamente después de crear un usuario.
    2. El actor selecciona un usuario desde la vista Usuarios (EA1)
    3. El actor selecciona mostrar desde la vista Administrar usuarios (EA2)
  3. Se observa la siguiente información de un usuario:
    1. Id
    2. Usuario
    3. Nombres
    4. Apellidos
    5. Correo electrónico
    6. Roles
    7. Activo
Extensiones del Escenario Alternativo:
  1. El sistema únicamente muestra los usuarios activos, en caso de no existir, se desplegará un mensaje informando que no se encontraron resultados.
Notas y Preguntas:
  • No se permite la actualización directa de la información desde esta vista.

CU-03: Actualizar usuario

Resumen:  El actor actualiza la información de un usuario.
Prioridad: Opcional
Frecuencia de Uso: Rara vez
Actores Directos: El administrador del sistema
Prerequisitos: Se debe haber iniciado sesión con un usuario con rol de administrador.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo Usuarios.
  2. El actor ingresa a la opción Administrar usuarios.
  3. El actor selecciona la opción actualiza.
  4. El sistema despliega la siguiente información del usuario permitiendo la actualización. (EA1)
    1. Usuario (EA2,EA3)
    2. Nombres (EA4)
    3. Apellidos (EA5)
    4. Correo electrónico (EA6)
    5. Roles (EA7)
    6. Activo
  5. El actor envia el formulario.
  6. El sistema despliega la información capturada.(EA8)
Extensiones del Escenario Alternativo:
  1. Si el usuario ha sido relacionado con alguna operación, se impide la actualización del usuario y se desplega un mensaje mencionando que se contacte al administrador del sistema.
  2. El usuario no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 3 y 20, respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardará el usuario.
  3. Si el usuario es actualizado y existe, no se guardará, se regresará al formulario de captura y se desplegará el mensaje de error.
  4. Los nombres no pueden ser vacíos o únicamente contener caracteres vacíos, de ser así, el sistema desplegará el mensaje y no guadará el usuario.
  5. Los apellidos no pueden ser vacíos o únicamente contener caracteres vacíos, de ser así, el sistema desplegará el mensaje y no guadará el usuario.
  6. El correo electrónico no puede ser vacío o únicamente contener caracteres vacíos y debe ser un correo válido, de lo contrario el sistema desplegará un mensaje informando el error y no guardará el usuario.
  7. Si no se selecciona al menos un rol, el sistema no guardará el usuario e indicará el error en el formulario.
  8. Si ocurre algún error al enviar el formulario, el rol no se guardará, ni redireccionará a la otra vista donde se muestra la información capturada.
Notas y Preguntas:
  • Se omite actualizar la contraseña del usuario.

CU-08: Crear rol

Resumen: El actor navega al módulo Roles y crea el rol con los campos solicitados.
Prioridad: Esencial
Frecuencia de uso: Rara vez
Actores Directos: Administrador del sistema
Prerequisitos: Se debe haber iniciado sesión con un usuario con rol de administrador.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo Roles.
  2. El actor ingresa a la opción crear rol.
  3. El sistema muestra el formulario para capturar los datos del rol.
  4. El actor captura los campos del formulario:
    1. Rol (EA1,EA2)
    2. Url inicial (EA3,EA4)
    3. Prioridad (EA5)
    4. Activo
  5. El actor envia el formulario.
  6. El sistema despliega la información capturada. (EA6)
Extensiones del Escenario Alternativo:
  1. El rol no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 45, respectivamente. En caso de no cumplir con lo requerido se desplegará un mensaje con el error correspondiente en el formulario y no se guardará el rol.
  2. Si el rol ya existe no se guardará, se regresará al formulario de captura y se desplegará el mensaje indicado.
  3. La Url inicial no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100 respectivamente. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardará el rol.
  4. Asegurarse de introducir la url inicial correctamente para evitar que posteriormente los usuarios con el rol a crear sean direccionados a una página no existente.
  5. La prioridad debe ser un número entero, en caso de no serlo, el sistema desplegará un mensaje de error y no guardará el rol.
  6. Si ocurre algún error al intentar enviar el formulario, no se guardará el usuario, ni redireccionará a la otra vista donde se muestra la información ingresada.
Notas y preguntas:
  • Se debe haber iniciado sesión con un usuario con rol de administrador.
  • El activo está seleccionado por defecto para automáticamente esté activado el rol a crear.
