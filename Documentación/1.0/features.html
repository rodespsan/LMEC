F-00: Administración de usuarios

Prioridad: Esencial
Esfuerzo: Horas
Riesgo: Seguro
Área(s) Funcional(es): Gestión de usuarios
Caso(s) de Uso: CU-00 CU-11
Descripción: La administración de usuarios permitirá:
  • Crear usuarios
  • Eliminar usuarios
  • Actualizar usuarios
  • Listar usuarios
Los usuarios contendrán los siguientes campos:
  • Usuario (entre 3 y 20 caracteres)
  • Contraseña (entre 8 y 20 caracteres)
  • Nombres
  • Apellidos
  • Correo electrónico
Notas y Preguntas:
  • La contraseña debe encriptarse en la base de datos usando MD5
  • ¿La contraseña deberá forzar a usar caracteres alfanuméricos y no alfanuméricos?

F-01: User registration

Priority: Essential
Effort: Days
Risk: Safe
Functional area(s): Administration
Use case(s): UC-01
Description: Visitors can come to the site and register themselves. They must provide the following information:
  • username
  • email address (twice to catch typos)
  • real name

Precise Details:

  • username must be unique (not equal to any other existing user name)
  • username must be of the form [a-zA-Z0-9]{2,16} and is not case sensitive
  • email address must be of the form [-a-zA-Z0-9_.]{2,16}@[-a-zA-Z0-9_.]{6,64}
  • both entries of the email address must match
  • email address will be verified by sending the user's initial password there
  • real name must not be empty
  • leading and trailing spaces are stripped from all fields
