CU-01: Crear usuario

Resumen: El actor crea un usuario con los campos solicitados.
Prioridad: Esencial
Frecuencia de Uso: Rara vez
Actores Directos: Administrador del sistema
Prerequisitos: Se debe haber iniciado sesión con un usuario con rol de administrador.
Debe existir al menos un rol activo.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo de usuarios.
  2. El actor ingresa a la opción crear usuario.
  3. El sistema muestra el formulario para capturar los datos del usuario.
  4. El actor captura los campos del formulario:
    1. Usuario (EA1,EA2)
    2. Contraseña (EA3)
    3. Confirmar contraseña (EA4)
    4. Nombres (EA5)
    5. Apellidos (EA6)
    6. Correo electrónico(EA7)
    7. Roles (EA8)
    8. Activo
  5. El actor envia el formulario.
  6. El sistema guarda la información capturada. (EA9)
  7. El sistema despliega la información capturada.
Extensiones del Escenario Alternativo:
  1. Usuario no puede ser vacío o contener algún espacio, el número de caracteres mínimo y máximo son: 3 y 20, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y el usuario no se guarda.
  2. Si el usuario ya existe no se guarda, se regresa al formulario de captura y se despliega el mensaje de error.
  3. Contraseña no puede ser vacía o contener algún espacio, el número de caracteres mínimo y máximo son: 8 y 20, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente.
  4. Confirmar contraseña no puede ser vacía o contener algún espacio, o si no coincide con la contraseña, el sistema no guarda el usuario e indica el error correspondiente en el formulario.
  5. Los nombres no pueden ser vacíos o únicamente contener caracteres vacíos, de no cumplir lo requerido el sistema despliega el mensaje y no guarda el usuario.
  6. Los apellidos no pueden ser vacíos o únicamente contener caracteres vacíos, de no cumplir lo requerido, el sistema despliega el mensaje y no guarda el usuario.
  7. El correo electrónico no puede ser vacío o únicamente contener caracteres vacíos y debe ser un correo con formato válido de correo electronico, de lo contrario el sistema despliega un mensaje informando el error y no guarda el usuario.
  8. Si no se selecciona al menos un rol, el sistema no guarda el usuario e indica el error en el formulario.
  9. Si ocurre algún error al enviar el formulario, el usuario no se guarda, ni redirecciona a la vista donde se muestra la información capturada.
Notas y Preguntas:
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo se le impide al usuario el acceso al sistema pero existe el proceso de activación.
  • Los campos ( Usuario, Contraseña, Confirmar contraseña, Nombres, Apellidos, Correo electrónico, Roles ) son obligatorios.
  • La contraseña deberá encriptarse en la base de datos usando MD5.

Frecuencia de uso: Rara vez
Actores Directos: Administrador del sistema
Prerequisitos: Se debe haber iniciado sesión con un usuario con rol de administrador.
Escenario Principal de Éxito:
  1. El actor ingresa al módulo de roles.
  2. El actor ingresa a la opción crear rol.
  3. El sistema muestra el formulario para capturar los datos del rol.
  4. El actor captura los campos del formulario:
    1. Rol (EA1,EA2)
    2. Url inicial (EA3,EA4)
    3. Prioridad (EA5)
    4. Activo
  5. El actor envia el formulario.
  6. El sistema guarda la información capturada. (EA6)
  7. El sistema despliega la información capturada.
Extensiones del Escenario Alternativo:
  1. El rol no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 45, respectivamente. En caso de no cumplir con lo requerido se despliega un mensaje con el error correspondiente en el formulario y el rol no se guarda.
  2. Si el rol ya existe no se guarda, se regresa al formulario de captura y se despliega el mensaje indicado.
  3. La Url inicial no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100 respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guarda el rol.
  4. Asegurarse de introducir la url inicial correctamente para evitar que posteriormente los usuarios con el rol a crear sean direccionados a una página no existente.
  5. La prioridad debe ser un número entero, en caso de no serlo, el sistema despliega un mensaje de error y no guarda el rol.
  6. Si ocurre algún error al intentar enviar el formulario, no se guarda el rol, ni redirecciona a la otra vista donde se muestra la información capturada.
Notas y preguntas:
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo se le impide al rol su uso en el sistema pero existe el proceso de activación.
  • La prioridad más alta se establece con el número más alto.
  • Los campos: Rol, Url inicial y Prioridad son obligatorios.

CU-108: Crear refacción

Resumen: El actor crea una refacción con los campos solicitados
Prioridad: Esencial
Frecuencia de Uso: Algunas veces
Actores Directos: Administrador del sistema.
Recepcionista
Prerequisitos:

Se debe haber iniciado sesión con un usuario con rol de administrador o Recepcionista.
Debe existir al menos una marca activa.
Debe existir al menos un estado de refacción activo.
Debe existir al menos un proveedor activo.

Escenario Principal de Éxito:
  1. El actor ingresa al módulo de refacciones y selecciona la opción crear refacción
  2. El sistema muestra el formulario para capturar los datos de la refacción.
  3. El actor captura los campos del formulario:
    1. Marca (EA1)
    2. Estado de refacción (EA1)
    3. Proveedor (EA1)
    4. Nombre de la refacción (EA2)
    5. Número de serie
    6. Precio (EA3)
    7. Fecha de compra (EA4)
    8. Fecha de vencimiento de garantía (EA5)
    9. Número de factura
    10. Descripción
    11. Activo
  4. El actor envía el formulario.
  5. El sistema guarda la información capturada. (EA6)
  6. El sistema despliega la información capturada.
Extensiones del Escenario Alternativo:
  1. La marca, estado de refacción y proveedor no puede ser vacía. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados.
  2. El nombre de la refacción no puede ser vacío o únicamente contener caracteres vacíos, el número de caracteres mínimo y máximo son: 1 y 100, respectivamente. En caso de no cumplir con lo requerido se despliega el mensaje de error correspondiente en el formulario y no se guarda la refacción.
  3. El precio no puede ser vacío o únicamente contener caracteres vacíos y debe ser de formato monetario (7,2). En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados.
  4. La fecha de compra no puede ser vacía y debe tener el formato (AAAA-MM-DD). En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados.
  5. La fecha de vencimiento de garantía no puede ser vacía, debe tener el formato (AAAA-MM-DD) y debe ser mayor o igual que la fecha de compra seleccionada. En caso de no cumplir con lo requerido se desplegará el mensaje de error correspondiente en el formulario y no se guardan los cambios realizados.
  6. Si ocurre algún error al enviar el formulario, no se guarda la refacción, ni se redirecciona a la otra vista donde se muestra la información ingresada.
Notas y Preguntas:
  • El campo activo por defecto inicia seleccionado.
  • Al desactivar el campo activo, la refacción no se visualiza para otros usuarios del sistema pero existe el proceso de activación.
  • Los campos (Marca, Estado de refacción, Proveedor) despliegan los registros activos.
  • Los campos (Fecha de compra, Fecha de vencimiento de garantía) despliegan un calendario que permite seleccionar la fecha deseada.
  • Los campos (Marca, Estado de refacción, Proveedor, Nombre de refacción, Precio, Fecha de Compra, Fecha de vencimiento de garantía) son obligatorios y los campos (Número de serie, Número de factura, Descripción, Activo) no son obligatorios.
