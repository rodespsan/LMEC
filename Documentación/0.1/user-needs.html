Propuesta del proyecto 
Especificación de requerimientos de software

Objetivos Acordados

    Proveer un sistema al Laboratorio de Mantenimiento de Equipo de Cómputo que se ajuste a las necesidades actuales del mismo, accesible por múltiples usuarios desde múltiples plataformas, fácil de actualizar y mejorar, y que garantice la integridad, confidencialidad y disponibilidad de la información.

    • El sistema debe ser multiusuario.
    • El sistema debe ser accesible.
    • El sistema debe ser multiplataforma.
    • El sistema debe ser seguro.
    • El sistema se debe ajustar a las necesidades actuales del Laboratorio.
    • Se liberarán versiones incrementales para su uso y evluación hasta alcanzar la versión final del proyecto.


Entorno

Entorno general

El sistema será usado por el personal del Laboratorio de Mantenimiento de Equipo de Cómputo. Actualmente laboran 3 profesionistas del área de TI: el responsable del departamento, el responsable del mantenimiento preventivo, el responsable del mantenimiento correctivo; y el personal de apoyo secretarial. También se cuenta con un número variable de alumnos que realizan sus practicas profesionales o servicio social que apoyan a los técnicos en distintas actividades dentro del departamento.

El personal de apoyo secretarial es quien realiza las siguientes actividades:

  • Recepción y entrega de equipos de cómputo.
  • Recepción e inventario de refacciones.
  • Altas, bajas y cambios de los catálogos del sistema.

Los responsables de mantenimiento de preventivo y correctivo son quienes realizan las siguientes actividades:

  • Diagnósticos
  • Mantenimientos preventivos
  • Mantenimientos correctivos

El responsable del departamento es quien realiza las siguientes actividades:

  • Supervisa las actividades del resto del personal del departamento.
  • Asigna ordenes de equipo a los técnicos.
  • Asigna las refacciones a las ordenes.
  • Verifica los trabajos realizados a los equipos.
  • Obtiene estadísticas del desempeño de sus procesos.

Entorno tecnológico

El sitema será una aplicación Web que utiliza la arquitectura cliente-servidor para que sea accesible por múltiples usuarios remotamente.

Podrá ser usado desde cualquier sistema operativo, usando los navegadores más comunes (Internet Explorer, FireFox y Chrome) en sus versiones más recientes.

Se ejecutará utilizando el softaware:

  • sistema operativo: Ubuntu Server Linux
  • servidor Web: Apache HTTP Server
  • lenguaje de programación: PHP
  • servidor de base de datos: MySQL

Entorno físico

La aplicación Web se encontrará ubicada en la Intranet en el área de servidores de la Facultad, la cual cuenta con aire acondicionado las 24 horas del día. Los servidores se encuentran funcionando 24/7 a excepción de los tiempos de mantenimiento generalmente realizados en períodos vacacionales. El área cuenta con UPS de respaldo en caso de cortes de energía de hasta de 20 minutos.

El servidor Intranet cuenta con:

  • doble fuente de poder para evitar pérdida de corriente.
  • configuración de arreglo de disco duro RAID 1.
  • sistema de respaldo diario a cinta.


Partes Interesadas / Actores

Administrador: Persona que tiene todos los permisos en el sistema, y que puede otorgar permisos a otros usuarios.
Recepcionista: Pesona que tiene privilegios para administrar los catálogos del sistema y capturar entradas y salidas de servicios.
Técnico: Personal técnico que se encarga de realizar los servicios y tiene privilegios para dar seguimiento a estos.

Necesidades de Rendimiento y Capacidad

Se requiere que el sistema soporte:

  • Mínimo 10 usuarios concurrentes.
  • Más de 1000 accesos diarios a la base de datos (agregar, consultar o modificar registros).
  • 1Gb de disco duro en el servidor para almacenar la información generada por el departamento.
